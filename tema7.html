Control de Acceso

El control de acceso es un mecanismo de seguridad que regula quién puede acceder a los archivos y directorios dentro de un sistema operativo y qué tipo de operaciones pueden realizar. Su objetivo es proteger la integridad, confidencialidad y disponibilidad de la información almacenada.

Existen tres enfoques principales para el control de acceso en los sistemas operativos:

  • Control de Acceso Discrecional (DAC): El propietario de un archivo o directorio decide quién puede acceder y qué permisos asignar. Este se basa en listas de control de acceso (ACL - Access Control List).
  • Control de Acceso Obligatorio (MAC):Las reglas de acceso están definidas por el sistema y no pueden ser modificadas por los usuarios. Se usa etiquetas de seguridad para clasificar datos y restringir accesos.
  • Control de Acceso Basado en Roles (RBAC): Los permisos se asignan según el rol del usuario dentro de la organización.

En Métodos de Implementación de Control de Acceso tenemos:

  • Matrices de Control de Acceso: Es una tabla donde las filas representan usuarios y las columnas representan archivos. Cada celda define los permisos como la lectura, escritura, ejecución, etc. Puede volverse ineficiente en sistemas con muchos usuarios y archivos.
  • Listas de Control de Acceso (ACLs): En donde cada archivo/directorio tiene una lista de usuarios autorizados con permisos específicos.
  • Listas de Capacidades: En lugar de asociar permisos a archivos, se asignan a usuarios específicos y siendo similar a una ACL, pero desde la perspectiva del usuario en lugar del archivo.

Control de Acceso en Sistemas Operativos Específicos

  • Linux/UNIX: Control de acceso basado en permisos de usuario, grupo y otros (rwx). Uso de ACLs y SELinux para seguridad avanzada.
  • Windows: Uso de Access Control Entries (ACEs) en sistemas NTFS y implementa Windows User Account Control (UAC) para restringir accesos administrativos.
  • MacOS: Basado en listas de control de acceso avanzadas y sandboxing para aplicaciones.

El control de acceso es un componente esencial en la seguridad informática. Los sistemas operativos modernos implementan diferentes enfoques para garantizar que los archivos y directorios sean accesibles solo por usuarios autorizados, evitando accesos no deseados y protegiendo la información crítica.

Sistemas FAT

El sistema de archivos FAT (File Allocation Table) es uno de los primeros y más utilizados en la historia de los sistemas operativos. Fue desarrollado por Microsoft para MS-DOS y ha sido ampliamente utilizado en discos duros, unidades USB y tarjetas de memoria debido a su simplicidad y compatibilidad con múltiples sistemas operativos.

El sistema FAT organiza los archivos en una estructura jerárquica con tres componentes principales siendo:

  • Boot Sector (Sector de Arranque): Contiene información clave sobre el sistema de archivos, como el tamaño del clúster y la ubicación de la FAT.
  • File Allocation Table (FAT): Es una tabla que registra qué bloques de almacenamiento pertenecen a qué archivos. Funciona como una lista enlazada que permite rastrear la ubicación de los archivos en el disco.
  • Directorio Raíz: Contiene información sobre los archivos y subdirectorios almacenados, incluyendo nombres, atributos y direcciones de inicio en la FAT.

Existen tres versiones principales del sistema FAT, cada una con diferentes capacidades y limitaciones, los cuales son:

  • FAT12 (12-bit): Soportando discos de hasta 16 MB y usado en disquetes y sistemas embebidos.
  • FAT16 (16-bit): Soportando discos de hasta 2 GB y fue el estándar en MS-DOS y versiones antiguas de Windows.
  • FAT32 (32-bit): Soportando discos de hasta 2 TB y mejora la eficiencia en la gestión del espacio reduciendo el tamaño del clúster.

Las ventajas que nos otorga el sistema FAT son:

  • Simplicidad y bajo uso de recursos.
  • Compatible con la mayoría de sistemas operativos, incluyendo Windows, macOS y Linux.
  • Ideal para dispositivos extraíbles como USB y tarjetas SD.

    • En desventajas tenemos que:

  • Falta de características avanzadas como permisos de acceso y journaling.
  • Fragmentación de archivos, lo que reduce la eficiencia.
  • Límite de tamaño de archivos en FAT32 (máximo 4 GB).

El sistema FAT sigue siendo relevante hoy en día debido a su compatibilidad y facilidad de uso. Sin embargo, ha sido reemplazado en muchos casos por sistemas más modernos como NTFS y exFAT, que ofrecen mejor seguridad y gestión de almacenamiento.
